# 新文章创建技能

基于现有文章的格式和结构，创建符合博客标准的新文章。

## 使用方法

当需要创建新文章时，运行此技能来生成符合以下标准的文章模板：
- 正确的 front matter 格式
- 统一的文章结构
- 符合博客规范的 markdown 格式

## 文章规范

### Front Matter 结构

```yaml
---
layout: post
title: 文章标题
subtitle: 文章副标题
categories: [分类1, 分类2]
tags: [标签1, 标签2, 标签3]
date: YYYY-MM-DD 00:00:00 +0800
---
```

### 文章结构模板

1. **标题**：使用 H1 标题，包含完整文章标题
2. **引言段落**：简要介绍文章内容和价值
3. **主体内容**：
   - 使用 H2 作为主要章节标题
   - 使用 H3 作为子章节标题
   - 代码块使用语言标识
   - 表格用于对比展示
4. **总结部分**：使用 `---` 分隔线
5. **检查清单**：使用 markdown checkbox 格式

### 格式要求

- **代码块**：使用三个反引号 + 语言标识
- **行内代码**：使用单个反引号
- **强调**：使用 `**粗体**` 或 `*斜体*`
- **表格**：使用标准 markdown 表格格式
- **列表**：使用 `-` 或 `1.` 格式
- **分隔线**：使用 `---`

### 内容风格

- **技术导向**：注重实用性和可操作性
- **步骤清晰**：使用编号步骤说明操作流程
- **示例丰富**：提供具体的命令和配置示例
- **问题解决**：包含常见问题和解决方案
- **对比分析**：使用表格对比不同方案

## 分类和标签规范

### 常用分类

- `Development` - 开发相关
- `DevOps` - 运维部署
- `Shell` - Shell 配置
- `Python` - Python 开发
- `Package Management` - 包管理
- `Container` - 容器技术
- `Linux` - Linux 系统
- `Browser` - 浏览器
- `Geek` - 极客装备
- `Rust` - Rust 语言

### 标签命名

- 使用小写字母和连字符
- 包含具体的技术名称
- 添加相关工具名称
- 包含应用场景标签

## 文件命名规范

文件名格式：`YYYY-MM-DD-article-slug.md`

其中：
- `YYYY-MM-DD`：发布日期
- `article-slug`：文章标题的英文 slug，使用连字符分隔

## 质量检查清单

在发布文章前，确保：

- [ ] Front matter 格式正确
- [ ] 所有代码块都有语言标识
- [ ] 链接和图片路径正确
- [ ] 表格格式规范
- [ ] 检查清单完整
- [ ] 文件命名符合规范
- [ ] 分类和标签准确
- [ ] 内容结构清晰

## 使用流程

1. 确定文章主题和目标受众
2. 选择合适的分类和标签
3. 使用标准模板创建文章
4. 按照结构规范编写内容
5. 运行质量检查清单
6. 保存到 `_posts` 目录

此技能确保所有新文章都符合博客的统一标准，提供一致的阅读体验。
